# G13 Author/Expert Walkthrough Template

Use this template only after a real author or expert performs the walkthrough.
Do not fill it with generated dry-run data, agent output, or inferred timing.

## Files To Produce

```text
reports/g13_author_expert_walkthrough.md
reports/g13_author_expert_walkthrough_observations.csv
reports/g13_author_expert_walkthrough_command_log.csv
```

## Report Template

Copy the block below into `reports/g13_author_expert_walkthrough.md` and fill
the values from the real walkthrough.


## Local Validation Command

After the three G13 files are filled, run:

```bash
make g13-check
```

The validator is intentionally strict. It checks that the evidence is signed,
has both conditions and all T1-T8 tasks, points to existing portable evidence
paths, and avoids positive hardware/vendor-tool validation claims. It is a
required gate, but it is not the only gate: G13 still needs the full closure
rerun and read-only subagent re-review before any final status update.

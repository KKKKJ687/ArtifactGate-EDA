## G13 Author/Expert Walkthrough Protocol

Use this protocol only when a real author or expert performs the review. Do not
fill it with generated or inferred results.

### Required Inputs

- Evaluator name or anonymized evaluator ID.
- Evaluator role, such as author, EEE/CS expert, artifact-reviewer, or
  senior student with HDL/SPICE familiarity.
- Review date in UTC.
- Whether this is a single expert walkthrough or a participant study.
- Confirmation that timing values are measured during the walkthrough if timing
  is reported.
- Confirmation that no hardware, Vivado timing, DFX deployment, bitstream, or
  board-validation claim is being made from this walkthrough.
- Evaluator attestation/signoff statement. For this packet, signoff means a
  typed evaluator name or anonymized evaluator ID, evaluator role, UTC date, and
  the statement that the reviewer performed the listed tasks on the listed
  materials. It does not imply ethics approval, participant-study approval, or
  hardware validation.

### Preparation Commands

Run these before the human/expert review:

```bash
make rq10-reviewer-walkthrough
make ist-package
make ist-strong-l2
```

Record the command output and UTC timestamps in the walkthrough log. The
existing generated dry-run command log is:

```text
reports/rq10_reviewer_walkthrough_command_log.csv
```

### Walkthrough Tasks

The reviewer should inspect both conditions:

| Condition | Material |
| --- | --- |
| Manual package | README, zip/package files, checksum or manifest material |
| ArtifactGate package | ArtifactGate reports, claim ledger, manifest, and validation outputs |

Minimum current-file checklist for the two conditions:

| Condition | Required files or paths to inspect |
| --- | --- |
| Manual package | `README.md`; `docs/ist_stronger_plan_source_record.md`; `release/artifactgate_eda_ist_evaluation_artifacts.zip`; `paper/MANUSCRIPT_REPRO_PACKAGE.md`; `paper/manuscript_artifact_manifest.csv` |
| ArtifactGate package | `reports/IST_GAP_AUDIT.md`; `reports/IST_VERIFICATION_RECEIPTS.json`; `reports/IST_WORKFLOW_GOVERNOR_GATE_LEDGER.md`; `reports/ist_manuscript_claim_gate.md`; `reports/claim_boundary_scan.csv`; replay output directories under `outputs/` generated by the Make targets |

Use the same task family as RQ10:

| Task | Decision |
| --- | --- |
| T1 | Identify whether a required log or artifact is missing. |
| T2 | Identify whether a hash mismatch is present. |
| T3 | Identify whether a non-portable local path is present. |
| T4 | Identify whether a hardware-boundary claim is unsupported. |
| T5 | Identify whether a Yosys-to-Vivado evidence escalation is unsupported. |
| T6 | Locate replay instructions and decide whether replay commands are available. |
| T7 | Locate the claim-evidence table and decide whether claims are traceable. |
